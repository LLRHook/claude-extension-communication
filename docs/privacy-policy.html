Blade Bridge - Privacy Policy

Last updated: March 22, 2026

Summary

Blade Bridge does not collect, store, transmit, or share any user data. It is a local-only communication bridge between your terminal and your browser.

Data Collection

Blade Bridge collects no data whatsoever. Specifically:

  • No personal information is collected
  • No browsing history is collected
  • No analytics or telemetry are sent
  • No cookies are used
  • No network requests are made to external servers

How It Works

Blade Bridge communicates exclusively between your local CLI and the Chrome extension using Chrome's Native Messaging protocol. All communication happens locally on your machine via stdin/stdout pipes and a local IPC socket. No data leaves your computer.

Permissions

  • nativeMessaging - Required to communicate with the local CLI via Chrome's Native Messaging protocol.
  • tabs - Required to list, open, close, and navigate browser tabs when commanded by the CLI.
  • scripting - Required to execute scripts on pages when commanded by the CLI.
  • activeTab - Required to interact with the currently active tab.
  • debugger - Required to execute JavaScript on pages that have strict Content Security Policies.
  • host_permissions (<all_urls>) - Required so the CLI can interact with any tab regardless of the URL.

Third Parties

No data is shared with any third party. Blade Bridge has zero external dependencies and makes zero network requests.

Changes

If this policy changes, the update will be posted here with a new date.
